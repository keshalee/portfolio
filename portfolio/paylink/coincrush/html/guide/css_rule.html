HTML

Syntax

CSS

Syntax

  • indent : 탭(tab)을 사용
  • 여러 셀렉터를 하나에 쓸 경우, 한 중에 하나의 셀렉터 사용
  • {앞뒤에 한칸 띄우지 않음}
  • :뒤에 한칸 띄우지 않음
  • 콤마(,) 다음에 한칸 띄우기
  • rgb(), rgba(), hsl(), hsla(), rect()안의 콤마(,)는 띄어쓰기 하지 않음
  • 소수점 0으로 시작하는 value는 0을 생략 0.5대신 .5사용
  • 속성 값이 '0'이면 단위를 생략
  • 색상값은 단축 표기하지 않음. (IE의 Gradation을 적용할 경우 색상값이 적용되지 않는 경우가 있음)

선언순서

선언의 순서는 아래와 같이 그룹핑

  1. 1. 위치관련(position, top, right, bottom, left, z-index)
  2. 2. 박스모델 관련(display, float, width, height, margin, padding)
  3. 3. 타이포그래피(font, line-height, color, text-align)
  4. 4. 시각효과(background, border)
  5. 5. 기타(opacity)

id/class 사용

레이아웃을 제외하고 가급적 id 사용은 지양

css 파일 구조

  • common.css: html 기본 속성값 초기화/공통 설정값(H tag, Button, Form, table, ul list)
  • layout.css: 사이트 레이아웃 요소
  • main.css: 메인 전용
  • product.css: 상품목록 및 상세
  • order.css: 주문/결제 관련
  • member.css: 회원가입/로그인/마이페이지
  • customer.css: 고객센터
  • popup.css: 팝업관련

id/class 명명 규칙

  • -(하이픈)은 가능하면 사용하지 않고 _(언더바)를 사용한다.
  • id는 camelcase 방식을 사용하고 class는 underscore 방식을 사용한다.
    ex) id="boxWrap" class="small_box"
  • id, class naming 은 가능하면 중복되지 않게 생성한다.
  • naming은 형태_의미_상태 순서로 조합하며 3단계를 넘어가지 않도록 한다.
    ex) btn_modify_on, box_info, dot_list
  • 거의 같은 형태에서 미묘한 변경이 있는 경우 다중 class를 적용한다
    ex) class="bullet_list" class="bullet_list type2"

Image

image 폴더 구조

  • common: header, footer영역과 같이 공통으로 사용되는 이미지
  • main: 메인에 사용되는 이미지
  • product: 상품목록 및 상세에 사용되는 이미지
  • order: 주문/결제 관련에 사용되는 이미지
  • member: 회원가입/로그인/마이페이지에 사용되는 이미지
  • customer: 고객센터에 사용되는 이미지
  • temp: 실제 반영되지 않을 샘플 이미지

image 명명 규칙

  • -(하이픈)은 가능하면 사용하지 않고 _(언더바)를 사용한다.
